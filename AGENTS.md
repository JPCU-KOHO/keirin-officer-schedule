# AI作業指示書

このリポジトリは、支部長等の出走・参加予定を公開する GitHub Pages サイトです。作業前に必ず [docs/MAINTENANCE.md](docs/MAINTENANCE.md) を読み、そこにある公開・確認手順に従ってください。

## 作業の原則

- 変更対象は `main` ブランチです。公開物は GitHub Actions が生成するため、ローカルの `public/` や `tmp/` をコミットしません。
- 名簿の Excel ファイルは公開済みの運用データです。ただし、名簿・個人名を新たに公開する変更は、管理者の明示的な承認がある場合だけ行います。
- トークン、Cookie、認証情報、Actions の実行ログに含まれる秘密情報は、ファイル・Issue・チャットへ書き出しません。個人用アクセストークンは不要で、保管も禁止です。
- 既存の未追跡ファイルや他者の差分を消さず、今回の作業に関係するファイルだけを変更・コミットします。
- `git reset --hard`、強制 push、履歴改変は行いません。

## 主な変更箇所

|目的|ファイル|
|---|---|
|名簿読込・出走状況・予定取得|`work/keirin_status.py`|
|Excel監査データからサイト用JSONへ変換|`work/export_keirin_status_json.py`|
|画面・地区タブ・対象日選択|`work/render_static_site.py`|
|毎朝の取得・Pages公開・障害通知|`.github/workflows/update-and-deploy.yml`|
|詳細な運用手順|`docs/MAINTENANCE.md`|

## AIへの依頼を実装する手順

1. 依頼内容に対応する上記ファイルを確認し、既存の例外・注意書き・自動更新を壊さない設計にする。
2. Pythonの構文確認と、必要に応じて静的HTML生成を行う。
3. 変更を限定してコミット・pushする。
4. GitHub Actions の **Update and deploy schedule** を手動実行し、`build` と `deploy` の成功を確認する。
5. 公開サイトを開き、依頼された表示・データを確認する。Pages反映直後に古い画面が残る場合は、数秒待って再読込する。

## 変更時の重要事項

- 常勤役員の除外や役職名の例外は、画面側ではなく `keirin_status.py` の定数で管理します。これにより毎朝の自動更新でも維持されます。
- 地区順は `render_static_site.py` の `REGIONS` が正です。九州は `福岡 → 佐賀 → 長崎 → 大分 → 熊本 → 鹿児島` の順です。
- 対象日を選んだ将来予定は「前検日」「参加予定」「予定なし」を表示します。将来のレース番号や開催区分は推測して表示しません。
- 予定情報には更新遅れがあり得るため、公開画面の注意書きを削除しません。
- 失敗通知、月次ハートビート、Pages設定を変更する場合は、`docs/MAINTENANCE.md` の「GitHub設定」を同時に更新します。
